# Whitepages Generator

## Описание логики работы генератора:
Проект представляет собой SaaS-решение, автоматизирующее процесс создания и публикации сайтов (Whitepages). Механика работы состоит из следующих этапов:
1. **Базовая настройка:** Пользователь выбирает язык (GEO, доступно 35+ вариантов) и тематику будущего сайта.
2. **Конфигурация:** Вводятся базовые данные проекта (название, домен) и выбирается визуальный стиль из заранее подготовленных дизайн-пресетов.
3. **Генерация контента (ИИ):** На основе заданных параметров бэкенд обращается к API нейросети (Gemini 1.5), которая автоматически пишет уникальные тексты, статьи и генерирует тематические изображения.
4. **Сборка проекта:** Система формирует готовый, адаптивный многостраничный сайт с правильной структурой, рабочими формами обратной связи, юридическими страницами (Privacy Policy, Terms of Use) и файлами sitemap.xml/robots.txt.

## План разработки и интеграции:
1. **Разработка ядра генератора:** Написание frontend и backend частей генератора с нуля согласно описанной логике.
2. **Сбор и настройка доступов:** Получение необходимых сервисных данных и API-ключей от Keitaro и Google Search Console.
3. **Интеграция с Keitaro (Автозалив):** Написание модуля для автоматической выгрузки готовых сайтов напрямую в трекер в качестве локальных лендингов.
4. **Интеграция с Google Search Console:** Разработка скрипта для автоматического добавления сайтов в панель GSC и отправки sitemap на индексацию.
5. **Подключение Gemini API:** Приобретение и интеграция ключа Gemini 1.5. Нейросеть необходима для качественного создания текстового и визуального контента, чтобы сайты получались полностью готовыми к заливу трафика.

## Оценка сроков:
На реализацию полного цикла закладывается **1-2 недели**:
* **1-я неделя:** Написание основного кода, создание логики работы и интеграция всех API.
* **2-я неделя:** Комплексное тестирование всех этапов, отладка автозалива и доведение инструмента до стабильного production-состояния.
